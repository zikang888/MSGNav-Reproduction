问题背景：零样本具身导航

Zero-Shot Embodied Navigation

核心挑战
在不依赖特定物体类别训练数据的前提下，让智能体仅通过自然语言描述或参考图片，就能在未知的 3D 环境中找到任意目标物体。

传统方法的局限

  • 需要针对每个目标物体类别进行大量训练（数百万帧强化学习）
  • 无法泛化到训练集之外的物体类别
  • 面对新环境和新目标描述时完全失效

零样本导航要求

1. 开放集视觉感知
识别任意物体，不限于预定义类别
2. 3D 空间理解
构建和维护环境的 3D 空间表征
3. 语义对齐
将自然语言/图像描述与 3D 场景对应

MSGNav 核心创新

多模态 3D 场景图作为连接 2D 感知与 3D 空间推理的桥梁

🔗
多模态场景图 (MSG)
将 YOLO-World + SAM + CLIP 的 2D 感知结果提升到 3D，构建包含物体空间位置、视觉特征、语义标签的有向图
✂️
关键子图选择 (KSS)
从数百个物体的场景图中智能选择与导航目标最相关的 Top-K 子图，压缩 VLM 输入
🔄
自适应词汇更新 (AVU)
VLM 可以请求对特定图像重新检测指定类别，解决"VLM 看到但检测模型漏检"的问题
🔁
闭环推理 (CLR)
将历史错误决策注入提示，避免 VLM 重复犯错
零样本特性
整个系统无需任何导航训练，仅依赖预训练模型（YOLO-World, SAM, CLIP）和 VLM API

算法原理：多模态 3D 场景图构建

Multimodal Scene Graph (MSG) - 动态维护的有向图 G = (V, E)

1
YOLO-World v8x 检测
输入 1280×1280 RGB，输出 N 个检测框 + 类别（200类HM3D），置信度≥0.5
2
SAM ViT-L 分割
生成精细像素掩码，过滤掩码面积<25px² 或 >50%图像面积
3
深度反投影
相机内参 + 深度图 → 3D点云，体素下采样 0.01m
4
DBSCAN 去噪
eps=0.1m, min_points=10，过滤点数<16的物体
5
CLIP 编码
6个视角渲染 → ViT-H/14 编码 → 1024维特征 + 零样本分类
6
跨帧匹配
空间IoU + 视觉余弦 → 匈牙利算法全局最优匹配
节点 V — 3D物体实例
• 3D 包围盒 (AABB) + 点云
• CLIP 视觉特征 (1024维)
• 物体类别名 + 置信度
• 检测次数 + 参考图像
边 E — 空间关系
• 中心距离 < 3.5m 时建立
• 关系图像（共现关键帧）
• 关系描述（VLM生成）
• 检测次数
周期性维护
每 20 步执行：merge（合并重叠度高且特征相似的物体）/ denoise（移除检测次数<3的低质量物体）

算法原理：关键子图选择与VLM决策

Key Subgraph Selection (KSS) + VLM Navigation Reasoning

关键子图选择 (KSS)

阶段一：语义预过滤
提取所有物体的类别名、房间标签、邻居关系 → VLM（仅文本）选出 Top-K=20 相关类别
阶段二：空间图剪枝
• 以关键物体为中心，保留直接邻居
• 贪心算法选择最少图像覆盖所有边
• 确保每个边至少有一张共现图像

VLM 导航决策

提示工程
任务指令 + 场景图摘要 + 当前观测 + 周边物体 + 历史信息 + 决策格式
三种决策输出
Object i：选择物体 i 作为目标 → 路径规划走向
Image i, category：指定类别重新检测（AVU）
Continue Exploration：继续探索前沿区域
终点确认
到达候选目标附近（1.5m）后，VLM 通过 6×40°=240° 环视进行 Yes/No 验证
技术参数
VLM: Qwen-VL-Max (阿里云 DashScope) | temperature=0.7, max_tokens=4096 | CLIP: ViT-H-14-quickgelu, dfn5b

项目总结

复现完成度、与论文对比、核心发现

环境搭建完成

Habitat-Sim 0.2.5, PyTorch 2.0.1+cu118, YOLO-World v8x, SAM ViT-L, CLIP ViT-H/14

数据准备完成

HM3D v0.2 val 场景, GOAT-Bench episodes, 模型权重下载 (共约3.1GB)

评估运行完成

5场景×38子任务, 1338张可视化PNG, 10个MP4导航视频

与论文差异分析

成功率68.42% vs 论文79.50%（-11.08%），可能原因：VLM差异(QLwen vs GPT-4o)、场景样本偏差、参数未调优

核心优势
• 真正的零样本，无需任何导航训练
• 图像引导表现优异 (80%)
• 场景图设计精巧，压缩VLM输入
• 模块化设计，易于替换或升级
当前局限
• 描述类任务仍是瓶颈 (53.85%)
• 大场景效率低 (SPL显著下降)
• 实时性不足 (~104s/步)
• 对VLM质量敏感
